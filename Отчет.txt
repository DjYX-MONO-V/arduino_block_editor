ОТЧЕТ О РАЗРАБОТКЕ: Arduino Block Editor
Версия: 0.0.21

1. ОПИСАНИЕ ПРОГРАММЫ
Arduino Block Editor — это специализированная среда разработки (IDE) для контроллеров Arduino, 
совмещающая визуальное редактирование функциональных блоков и написание кода. 
Программа ориентирована на удобство обучения и быструю разработку за счет интеграции вспомогательных инструментов.

2. РЕАЛИЗОВАННЫЙ ФУНКЦИОНАЛ
* Визуальный холст: Drag-and-drop блоков, масштабирование, панорамирование и сетка.
* Генератор кода: Автоматическое преобразование блоков в полноценный скетч (.ino) с учетом глобальных переменных, функций setup() и loop().
* Редактор кода: Подсветка синтаксиса (VS Code style), нумерация строк, подсветка текущей строки и поиск (Ctrl+F).
* Интеграция с Arduino CLI: Компиляция, загрузка прошивки, получение инфо о плате и запись загрузчика.
* Менеджер библиотек: Поиск, установка и удаление библиотек Arduino прямо из интерфейса.
* Продвинутый калькулятор: Режимы HEX/DEC/OCT/BIN, битовые операции и эмуляция разрядности контроллера (8/16/32/64 бит).
* PDF-Просмотрщик: Встроенное окно для чтения документации и учебников с поиском по тексту.
* Монитор порта: Чтение и отправка данных через Serial.
* Система тем: Темная тема оформления (VS Code Dark).
* Портативность: Возможность сборки в исполняемый файл (.exe) с иконкой.

3. ЧТО РАБОТАЕТ СТАБИЛЬНО
* Сохранение и загрузка проектов (.json).
* Экспорт проекта вместе со всеми зависимыми библиотеками в одну папку (Bundle).
* Синхронизация настроек путей к Arduino IDE и папке скетчей.
* Навигация по библиотеке блоков и сохранение пользовательских шаблонов блоков с превью.

4. ИЗВЕСТНЫЕ ОШИБКИ И ОГРАНИЧЕНИЯ
* PDF Просмотрщик (QWebEngine): 
    - В консоли могут появляться сообщения "js: Uncaught Error: Assertion failed". Это внутренние логи Chromium при отрисовке PDF, на работу программы не влияют.
    - Поиск в PDF может выдавать ошибку "getStrings of undefined", если вызван до полной инициализации JS-движка (добавлена задержка 500мс для минимизации).
    - Флаг циклического поиска (WrapsAround) отключен для совместимости со старыми версиями библиотек.
* Поиск в коде: Работает в пределах одного активного виджета или окна (нет сквозного поиска по всем блокам одновременно).
* Импорт примеров: Реализован базовый парсинг функций по регулярным выражениям. Сложные структуры C++ (классы, вложенные неймспейсы) могут импортироваться некорректно.

5. ПЛАНЫ ПО УЛУЧШЕНИЮ
* Реализация визуальных связей (линий) между блоками.
* Добавление кнопки вставки результата из калькулятора в код.
* Улучшение системы автодополнения кода в блоках.
* Возможность сворачивания (минимизации) блоков двойным кликом по заголовку.
